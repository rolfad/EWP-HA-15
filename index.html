Beschreibung git-Workflow

Zunächst wird mithilfe des git clone Befehls ein Projekt aus dem Repository auf die lokale Festplatte geklont und kann dann bearbeitet werden. Nach der Bearbeitung werden die veränderten Dateien mithilfe des git add Befehls zur Zwischenspeicherung vorgemerkt. Mit dem git commit Befehl werden dann alle vorgemerkten Dateien unter Angabe eines Kommetars (der möglichst hilfreiche und übersichtliche Angaben zu den erfolgten Änderungen gibt) zwischengespeichert (in lokale .git Datei). Schließlich werden die zwischengespeicherten Änderungen (commits) mit dem git push Befehl ins das Repository hochgeladen. Veränderungen an dem Projekt sind somit dokumentiert und lassen sich zeitlich nachvollziehen. Umgekehrt lassen sich mit dem git pull Befehl die lokalen Dateien auf den Stand der Dateien im Repository aktualisieren. Auf diese Weise kann effizient asynchron von mehreren Personen an einem Projekt gearbeitet werden, ohne 'neue aktuelle' Versionen aus verschiedenen Dateien manuell mergen zu müssen.

git clone https://github.com/rolfad/EWP-HA-15.git
git add index.html
git commit -m "Changed Tooltip Content"
git push
git pull
